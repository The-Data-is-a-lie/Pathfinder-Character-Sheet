# Grouping Related Stats

Players do not think in rulebook chapter order; they think in **tasks**. A
well-grouped sheet puts everything needed for a task in one visual neighborhood
so the eye does not bounce across the page mid-action.

## Proven groupings

| Cluster | Keep together |
| --- | --- |
| Identity | Name, class/level, race, alignment, size, speed (header strip) |
| Ability core | STR–CHA scores **and** modifiers, with room for temporary modifiers |
| Combat defenses | AC components, touch/flat-footed if shown, CMD, saves |
| Combat offense | BAB, weapons/attacks, CMB, damage, crit, attack conditionals |
| Resources | HP, temporary HP, conditions, limited-use trackers |
| Skills | Skill list with ability links; perception and initiative nearby if often used |
| Magic / powers | Spells known/prepared, spheres, Path of War readied list, DCs |
| Gear | Inventory next to encumbrance / capacity / wealth |
| Narrative | Biography, notes, traits flavor (demoted from combat chrome) |

## Pathfinder pitfalls

- **Splitting attack and damage** across distant panels forces double scanning during full attacks.
- **Ability scores without modifiers** force mental math; modifiers are the play surface.
- **Encumbrance far from inventory** breaks the “can I carry this?” task.
- **Conditionals** (feat riders, stance damage, spell buffs) belong next to the
  roll UI they modify, not only in a remote Buffs dump.

## Gestalt and cards

Use consistent section cards, headings, and rules (borders/dividers) so groups
read as **common regions**. In dense Pathfinder sheets, grouping quality matters
more than decorative flourish.
