# Typography for Stats

## Hierarchy ladder (recommended)

1. **Character name** — largest; small-caps or display serif is fine if legible.
2. **Key combat numbers** — ability mods, AC, attack bonuses, HP current — bold
   and large enough to read at arm’s length.
3. **Section titles** — clear but quieter than combat numbers.
4. **Field labels** — small caps or reduced size/dim color; never compete with values.
5. **Body / description HTML** — readable serif or sans at ~15–16px equivalent;
   long feat text is reference, not chrome.

## Practical notes

- Prefer the **modifier** as the hero number; raw ability score can be secondary.
- Test fillable/edit fields with **maximum length values** so layout does not
  overflow mid-session.
- Avoid ultra-thin weights on dark backgrounds (halation / lost strokes).
- Light-on-dark type often looks heavier than the same weight dark-on-light;
  adjust weight when dual-theming.
- Match rulebook *feel* with similar fonts when licensing allows; free
  alternatives are fine when commercial faces are unavailable.
